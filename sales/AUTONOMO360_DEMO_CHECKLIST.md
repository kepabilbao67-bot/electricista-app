# Autonomo360 — Checklist antes de ensenar la demo

## Preparacion (5 min antes)

- [ ] Abrir la app en el navegador (Chrome recomendado)
- [ ] Verificar que carga sin errores
- [ ] Tener credenciales de acceso a mano
- [ ] Preparar pantalla limpia (cerrar pestanas innecesarias)
- [ ] Si es presentacion en movil: abrir en Chrome movil o usar responsive del navegador

## Dashboard

- [ ] El panel muestra KPIs (con datos reales o modo demo)
- [ ] El grafico de facturacion se ve correctamente
- [ ] Las alertas y acciones rapidas funcionan
- [ ] No aparecen errores en consola

## Clientes

- [ ] La lista carga correctamente
- [ ] Se puede crear un cliente nuevo (usar datos ficticios para la demo)
- [ ] La busqueda funciona
- [ ] El detalle de un cliente muestra historial

## Partes de trabajo

- [ ] Se puede crear un parte nuevo
- [ ] Se anaden trabajos y materiales
- [ ] Los calculos (subtotal, IVA, total) son correctos
- [ ] Se puede guardar
- [ ] Se puede reabrir y editar
- [ ] La impresion muestra datos correctos
- [ ] No aparece "Modo demo" ni "Version demo"

## Presupuestos

- [ ] Se puede crear un presupuesto
- [ ] Se puede convertir en factura (si hay presupuestos existentes)
- [ ] Los importes son coherentes

## Facturas

- [ ] La lista de facturas carga
- [ ] Se puede crear una factura nueva
- [ ] Los estados (borrador, enviada, cobrada) funcionan
- [ ] Se puede eliminar un borrador

## Gastos

- [ ] Se puede registrar un gasto
- [ ] Las categorias aparecen correctamente

## Agenda

- [ ] El calendario muestra la semana actual
- [ ] Se puede crear una visita

## Movil (390px)

- [ ] Abrir menu hamburguesa
- [ ] Navegar entre secciones
- [ ] Crear un parte o factura desde movil
- [ ] No hay desbordamiento horizontal
- [ ] Los botones son pulsables

## Secciones que NO ensenar en demo inicial

- [ ] Comunicaciones (parcial — solo prepara mensajes)
- [ ] Leads (parcial — sin conversion completa)
- [ ] Exportar (funciona pero no es visualmente impactante)
- [ ] Normativa/Asistente (requiere clave OpenAI para respuestas completas)

## Despues de la demo

- [ ] Borrar datos ficticios creados durante la presentacion
- [ ] No dejar partes/facturas de prueba en produccion
- [ ] Anotar feedback del potencial cliente
- [ ] Enviar resumen por email si hubo interes

## Frases clave para la demo

- "Esto es una aplicacion real, en uso por un cliente"
- "Se adapta al negocio: colores, datos fiscales, catalogo de servicios"
- "Funciona en movil sin instalar nada"
- "Los datos estan en la nube, accesibles desde cualquier dispositivo"
- "Sistema en desarrollo activo — se puede ampliar con nuevas funciones"

## Frases que NO decir

- "Esta 100% terminada"
- "Cumple toda la normativa fiscal"
- "Funciona para cualquier autonomo sin adaptacion"
- "Tiene firma digital certificada"
- "Envia facturas electronicas obligatorias"
