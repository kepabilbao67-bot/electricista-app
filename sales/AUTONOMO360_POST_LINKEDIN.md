# Autonomo360 — Posts para LinkedIn

## Post 1: Presentacion del proyecto

Llevo meses desarrollando una herramienta de gestion para autonomos.

No es un concepto. Es una app real, en uso por un cliente del sector electrico.

Que hace:
- Gestiona clientes, presupuestos y facturas
- Registra partes de trabajo con materiales y firma
- Controla gastos y agenda
- Funciona desde el movil

Tecnologia: Next.js, React, TypeScript, Tailwind, Turso, Vercel.

No es un producto terminado para todos los autonomos. Es un sistema funcional que demuestra que se puede construir un SaaS vertical sin un equipo de 20 personas.

Si eres autonomo o conoces a alguien que gestiona su negocio con papeles y Excel, hablemos.

#SaaS #Autonomos #DesarrolloWeb #React #NextJS #KEPA

## Post 2: El parte de trabajo digital

Un electricista me pidio digitalizar sus partes de trabajo.

Antes: rellenaba a mano, perdia copias, no podia buscar trabajos antiguos.

Ahora:
- Crea el parte desde el movil
- Anade trabajos y materiales con precios
- El cliente firma en pantalla
- Imprime o guarda como PDF
- Todo queda guardado y es editable

No es magia. Es una app web con formularios bien hechos, base de datos real y un diseno pensado para usarse con una mano.

Lo construi como parte de Autonomo360, un sistema de gestion que estoy desarrollando para autonomos de servicios.

Si trabajas a domicilio y quieres dejar de perder papeles, esto es para ti.

## Post 3: Mostrar capacidad tecnica

Cuando un cliente te dice "quiero una app para gestionar mi negocio", la respuesta no es "imposible" ni "necesitas 200K de inversion".

La respuesta es: definir bien que necesita, construir lo minimo viable y iterar.

Esto es Autonomo360:
- 12 modulos funcionales
- Base de datos en la nube
- Auth, API REST, validaciones
- Responsive completo
- Desplegado en Vercel
- En uso real por un cliente

Stack: Next.js 16, React 19, TypeScript, Tailwind 4, Turso (SQLite distribuido).

Lo desarrollo yo solo como proyecto de KEPA tecnologIA.

No lo presento como producto final perfecto. Lo presento como prueba de que un desarrollador con criterio puede construir herramientas utiles para negocios reales.

Si necesitas digitalizar tu negocio o el de un cliente, hablemos.
